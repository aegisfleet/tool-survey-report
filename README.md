# ポちのツール情報収集サイト

![OGP画像](assets/images/default-ogp.png)

AIエージェントを用いてツールの情報を収集して公開するサイトです。

## 概要

このサイトは、**GoogleのJulesエージェント**が実施したツール調査レポートを効率的に公開・共有するためのプラットフォームです。各ツールの機能、特徴、使用方法を詳細に調査し、その結果をMarkdown形式のレポートとして作成・公開しています。これらのレポートは、ツールの内容把握と目的に応じた最適なツールの比較検討に役立てることを目的としています。Jekyll静的サイトジェネレーターを使用して自動的にWebサイトとして公開されます。

**🌐 サイトURL**: <https://aegisfleet.github.io/tool-survey-report/>

## 主な機能

- **📱 レスポンシブデザイン**: モバイル・デスクトップ対応
- **🔍 検索・フィルタリング**: リアルタイム検索とタグによるフィルタリング
- **🏷️ タグシステム**: カテゴリー別のレポート分類・管理
- **📊 統計表示**: 総レポート数・タグ数の表示
- **🔗 関連レポート**: タグベースの関連レポート自動表示
- **📄 レポートナビゲーション**: 前後のレポートへの移動機能
- **🎨 直感的UI**: 使いやすいカード型レイアウト
- **⚡ 高速表示**: Jekyll静的サイトによる高速ページ読み込み
- **🔧 SEO最適化**: メタタグ、OGP、構造化データ対応
- **♿ アクセシビリティ**: WCAG準拠のアクセシブルなデザイン
- **🤖 自動レポート更新**: Jules APIを使用した1日1回の自動レポート更新

## ローカル開発環境のセットアップ

### 前提条件

- Ruby 2.7以上
- Bundler gem
- Git

### インストール手順

1. **リポジトリのクローン**

   ```bash
   git clone <repository-url>
   cd tool-survey-report
   ```

2. **依存関係のインストール**

   ```bash
   bundle install
   ```

3. **ローカルサーバーの起動**

   ```bash
   bundle exec jekyll serve
   ```

4. **ブラウザでアクセス**

   ```text
   http://localhost:4000
   ```

### 開発用コマンド

- **開発サーバー起動（自動リロード付き）**

  ```bash
  bundle exec jekyll serve --livereload
  ```

- **ドラフトモードで起動**

  ```bash
  bundle exec jekyll serve --drafts
  ```

- **本番環境用ビルド**

  ```bash
  bundle exec jekyll build
  ```

- **設定変更を監視して起動**

  ```bash
  bundle exec jekyll serve --watch
  ```

## レポートの作成方法

### 1. 新しいレポートファイルの作成

`_reports/` ディレクトリに新しいMarkdownファイルを作成します：

```bash
touch _reports/tool-name.md
```

### 2. フロントマターの設定

ファイルの先頭にYAMLフロントマターを追加：

```yaml
---
title: "[ツール名] 調査レポート"
tool_name: "ツール名"
category: "カテゴリ名"
developer: "開発元企業名"
official_site: "公式サイトURL"
date: "YYYY-MM-DD" (作成日)
last_updated: "YYYY-MM-DD" (更新日)
tags:
  - "AI・機械学習"
  - "開発ツール"
  - "テスト・自動化"
description: "ツールの概要を1-2行で簡潔に記述"
layout: report
---
```

### 3. コンテンツの記述

フロントマターの後にMarkdown形式でレポート内容を記述。詳細な構成は `_reports/template.md` を参照してください。

**基本構成**:

1. 基本情報
2. 目的と主な利用シーン
3. 主要機能
4. 特徴・強み (Pros)
5. 弱み・注意点 (Cons)
6. 料金プラン
7. 導入実績・事例
8. サポート体制
9. 連携機能 (API・インテグレーション)
10. セキュリティとコンプライアンス
11. 操作性 (UI/UX) と学習コスト
12. ユーザーの声（レビュー分析）
13. 直近半年のアップデート情報
14. 類似ツールとの比較
15. 総評

## 実装済み機能の詳細

### ホームページ機能

- **リアルタイム検索**: レポートタイトルでの即座検索
- **タグフィルタリング**: ドロップダウンでのタグ別絞り込み

- **レスポンシブカード**: モバイル対応のカードレイアウト

### レポートページ機能

- **メタ情報表示**: 更新日・開発元・カテゴリ・公式サイト・タグ
- **パンくずナビ**: 階層構造の明確化
- **レポートナビゲーション**: 前後のレポートへの移動
- **関連レポート**: 共通タグによる関連レポート自動表示
- **構造化データ**: SEO対応のschema.org準拠

### SEO・OGP対応

- **動的メタタグ**: ページごとの最適化されたメタ情報
- **OGP画像**: SNSシェア用の画像設定
- **Twitter Card**: Twitter用の最適化
- **サイトマップ**: 自動生成されるXMLサイトマップ
- **RSSフィード**: 更新情報の配信

## 動作確認方法

### 基本的な動作確認

1. **サイトの起動確認**

   ```bash
   bundle exec jekyll serve
   ```

   エラーなく起動し、`http://localhost:4000` でアクセスできることを確認

2. **機能確認**
   - ✅ ホームページでレポート一覧が表示される
   - ✅ 検索ボックスでリアルタイム検索が動作する
   - ✅ タグフィルターでレポートが絞り込める
   - ✅ ソート機能で並び順を変更できる
   - ✅ 個別レポートページが正常に表示される
   - ✅ レポート間のナビゲーションが機能する
   - ✅ 関連レポートが表示される
   - ✅ タグクリックでフィルタリングされる

3. **レスポンシブデザイン確認**
   - ✅ ブラウザの開発者ツールでモバイル表示を確認
   - ✅ 異なる画面サイズで適切に表示される
   - ✅ タッチデバイスでの操作性を確認

### トラブルシューティング

**よくある問題と解決方法：**

- **Bundlerエラー**: `bundle install` を再実行
- **ポート競合**: `bundle exec jekyll serve --port 4001` で別ポートを使用
- **キャッシュ問題**: `bundle exec jekyll clean` でキャッシュをクリア
- **依存関係エラー**: `bundle update` で依存関係を更新
