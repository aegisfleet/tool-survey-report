# **Robot Framework 調査レポート**

## **1. 基本情報**

* **ツール名**: Robot Framework
* **開発元**: Robot Framework Foundation
* **公式サイト**: [https://robotframework.org/](https://robotframework.org/)
* **カテゴリ**: テスト/QA
* **概要**: 受け入れテスト、受け入れテスト駆動開発 (ATDD)、およびRPA（ロボティック・プロセス・オートメーション）のための、汎用的なオープンソース自動化フレームワークです。キーワード駆動型のアプローチを採用しており、技術者以外でも理解しやすいテストケースを作成できます。

## **2. 目的と主な利用シーン**

* **目的**: テスト自動化やRPAを、プログラミングスキルに依存せずに実現可能にすること。共通のキーワードを使用して、開発者、QA、ビジネス関係者が協力して自動化を推進することを目的としています。
* **主な利用シーン**:
  * WebアプリケーションのE2Eテスト (SeleniumLibrary, Browser Library等を使用)
  * モバイルアプリケーションのテスト (AppiumLibraryを使用)
  * デスクトップアプリケーションの自動化
  * RPAによる業務プロセスの自動化
  * APIテスト

## **3. 主要機能**

* **キーワード駆動テスト**: 自然言語に近い「キーワード」を使用してテストケースを記述できます。
* **高い拡張性**: PythonやJavaを使用して、独自のテストライブラリを簡単に作成・追加できます。
* **豊富なエコシステム**: Selenium, Appium, Playwright, Database, HTTPなど、多くの外部ツールやシステムと連携するためのライブラリが提供されています。
* **タグ付け機能**: テストケースにタグを付け、特定のカテゴリのテストのみを実行したり、スキップしたりできます。
* **レポートとログ**: 実行結果として、詳細なHTMLレポートとログファイルを自動生成します。

## **4. 特徴・強み (Pros)**

* **読みやすい構文**: テストケースが表形式や自然言語に近い形式で記述されるため、非技術者でも理解しやすく、メンテナンスが容易です。
* **柔軟な拡張性**: Pythonのエコシステムを活用でき、必要な機能をライブラリとして追加できます。
* **統合の容易さ**: CI/CDパイプラインへの組み込みが容易で、Jenkinsなどのツールと連携して自動実行できます。
* **オープンソース**: ライセンス費用がかからず、活発なコミュニティによって支えられています。

## **5. 弱み・注意点 (Cons)**

* **実行速度**: Pythonベースであり、キーワードの解析などのオーバーヘッドがあるため、純粋なコードベースのフレームワークと比較して実行速度が劣る場合があります。
* **デバッグの難易度**: キーワードの裏側で何が起きているかが見えにくいため、複雑な問題のデバッグが難しい場合があります。
* **学習コスト**: 基本的な使い方は簡単ですが、高度なカスタマイズや独自のライブラリ開発にはPythonの知識が必要となります。

## **6. 料金プラン**

* **オープンソース**: Apache License 2.0の下で無料で利用可能です。

## **7. 導入実績・事例**

* Nokia Networksで開発が始まった経緯もあり、通信業界をはじめ、金融、ヘルスケアなど、世界中の多くの企業で採用されています。
* 日本国内でも、大手SIerやテスト専門会社などで広く利用されています。

## **8. サポート体制**

* **ドキュメント**: 公式サイトおよびGitHubに詳細なユーザガイドやAPIリファレンスが提供されています。
* **コミュニティ**: Robot Framework Foundationが運営するForumやSlack、Stack Overflowなどで活発な議論が行われています。
* **カンファレンス**: 年次カンファレンス「RoboCon」が開催され、最新情報の共有が行われています。

## **9. 連携機能 (API・インテグレーション)**

* **ライブラリ**:
  * **SeleniumLibrary**: Webテスト用 (Selenium WebDriver使用)
  * **Browser Library**: 最新のWebテスト用 (Playwright使用)
  * **AppiumLibrary**: モバイルアプリテスト用
  * **RequestsLibrary**: APIテスト用
  * **DatabaseLibrary**: データベース操作用
* **CI/CDツール**: Jenkins, GitHub Actions, GitLab CIなど、主要なCI/CDツールと連携可能です。

## **10. セキュリティとコンプライアンス**

* オープンソースソフトウェアとして、コミュニティベースでセキュリティ修正が行われています。
* 企業内での利用においては、独自のセキュリティポリシーに準拠したライブラリの管理が必要です。

## **11. 操作性 (UI/UX) と学習コスト**

* **操作性**: 基本的にはテキストファイルでテストを記述しますが、RIDE (Robot Framework IDE) やVS Code拡張機能などを使用することで、GUIベースやコード補完を利用した効率的な開発が可能です。
* **学習コスト**: キーワード駆動であるため、初期の学習コストは低いですが、複雑なロジックを実装する場合はPythonの知識が必要となり、学習コストが上がります。

## **14. 類似ツールとの比較**

* **Cucumber**: BDD（振る舞い駆動開発）に特化したツールで、Gherkin構文（Given-When-Then）を使用します。Robot FrameworkもBDDスタイルをサポートしていますが、より汎用的なキーワード駆動が中心です。
* **Selenium**: Webブラウザ操作のライブラリであり、Robot FrameworkはSeleniumを利用するフレームワークという位置づけです（SeleniumLibrary経由）。
* **Gauge**: Markdownベースの仕様書を実行可能なテストにするツール。シンプルで読みやすい点がRobot Frameworkと共通していますが、よりドキュメント性を重視しています。

## **15. 総評**

* **総合的な評価**: Robot Frameworkは、技術者と非技術者の橋渡しをする強力な自動化フレームワークです。Web、モバイル、API、デスクトップなど、多様な対象を統一的なアプローチでテストできる点が大きな魅力です。
* **推奨されるチームやプロジェクト**:
  * テスト担当者がプログラミングに詳しくない場合や、ビジネス部門と連携してテストを進めたいチーム。
  * 複数のプラットフォーム（Web, Mobile, Desktop）にまたがる統合テストが必要なプロジェクト。
  * RPAとして業務自動化を行いたい場合。
