# **Claude Code 調査レポート**

## **1. 基本情報**

<!--
【ガイドライン】
- 必須: ツール名、開発元、公式サイト、カテゴリ、概要
- 概要は2-3文で簡潔に記述
-->

* **ツール名**: Claude Code
* **ツールの読み方**: クロード コード
* **開発元**: Anthropic
* **公式サイト**: [https://claude.ai/code](https://claude.ai/code)
* **関連リンク**:
  * ドキュメント: [https://docs.claude.com/en/docs/claude-code](https://docs.claude.com/en/docs/claude-code)
  * ベストプラクティス: [https://code.claude.com/docs/en/best-practices](https://code.claude.com/docs/en/best-practices)
  * Claude Code ベストプラクティス: [https://claude-code-best-practices.manus.space/](https://claude-code-best-practices.manus.space/)
* **カテゴリ**: 自律型AIエージェント
* **概要**: Anthropic社が開発した、ターミナル、IDE、Web、Slack上で動作する自律型コーディングエージェント。最新のClaude 4.5モデル（Opus/Sonnet/Haiku）を搭載し、自然言語での指示に基づいて、コードベースの探索、計画立案、コードの記述、テスト実行、Git操作までを自律的に行う。

## **2. 目的と主な利用シーン**

<!--
【ガイドライン】
- 3-5項目を箇条書きで記述
- 想定利用者を具体的に記載
-->

* **解決する課題**: 開発環境から離れることなく、AIに複雑なコーディングタスクや調査を任せ、コンテキストスイッチを減らし生産性を向上させる。
* **想定利用者**: ソフトウェアエンジニア、DevOpsエンジニア、テックリード。
* **利用シーン**:
  * 大規模なリファクタリングや機能追加の自動化
  * エラーログに基づいたデバッグと修正
  * 未知のコードベースの構造理解とオンボーディング
  * コミットメッセージの作成やPRの自動生成（GitHub/GitLab連携）
  * Slackからのタスク指示と実行

## **3. 主要機能**

<!--
【ガイドライン】
- 5-10項目の主要機能を箇条書きで記述
- 各機能は1-2文で概要を説明
-->

* **マルチプラットフォーム対応**: ターミナル（CLI）だけでなく、VS Code/JetBrainsの拡張機能、Webブラウザ、Slackアプリとして利用可能。
* **Agentic Search**: 手動でコンテキストを選択しなくても、エージェントが自律的にコードベースを探索・検索し、必要な情報を理解する。
* **Plan Mode**: 複雑なタスクに対して、実装前にコードベースを調査し、詳細な実行計画を立案・提示する。
* **強力な編集機能**: 依存関係や構造を理解した上で、複数のファイルにまたがる整合性の取れた変更を行う。
* **MCP (Model Context Protocol) 対応**: データベース、GitHub、Jiraなどの外部ツールと標準プロトコルで接続し、情報の取得や操作が可能。
* **安全な実行**: ファイルの変更やコマンド実行の前に必ずユーザーの許可を求める（設定により変更可能）。
* **CLAUDE.md**: プロジェクト固有のルールやコマンドを記述した設定ファイルを読み込み、文脈に沿った作業を行う。
* **LSP (Language Server Protocol) サポート**: コードの定義ジャンプ、参照検索、ホバー表示などのコードインテリジェンス機能を提供。
* **Claude in Chrome (Beta)**: ブラウザを直接制御し、UIの検証やWebベースのワークフローを実行可能。
* **プラグインシステム**: カスタムコマンド、エージェント、MCPサーバーなどをバンドルしたプラグインを導入し機能を拡張できる。

## **4. 開始手順・セットアップ**

<!--
【ガイドライン】
- アカウント作成から「Hello World」的な最初の動作確認までの手順
- 主要なコマンドや設定項目を記載

* **前提条件**:
  * Node.js (v18以降)
  * Anthropic API Key (またはClaude Proアカウント)
* **インストール/導入**:
  ```bash
  # npm経由でインストール (コマンド例)
  npm install -g @anthropic-ai/claude-code
  ```
* **初期設定**:
  * `claude login` コマンドで認証を行う。
  * 必要に応じて `CLAUDE_API_KEY` 環境変数を設定。
* **クイックスタート**:
  * プロジェクトディレクトリで `claude` コマンドを実行し、対話モードを開始する。

## **5. 特徴・強み (Pros)**

<!--
【ガイドライン】
- 3-5項目を箇条書きで記述
- 競合との差別化ポイントを明確に

* **最新モデルの搭載**: Claude 4.5 Opus/Sonnetを使用しており、非常に高い推論能力とコード生成品質を持つ。
* **自律性と制御のバランス**: 「調査→計画→実行→検証」という自律的なフローを持ちつつ、重要な場面ではユーザーの承認を求める安全設計。
* **環境適応性**: ターミナル派の開発者にはCLI、IDE派には拡張機能、モバイルや外出先ではWeb/Slackと、あらゆるシーンで利用できる。
* **深いコード理解**: 単なるファイルの中身だけでなく、プロジェクトの構造や依存関係を深く理解した上で作業を行う。

## **6. 弱み・注意点 (Cons)**

<!--
【ガイドライン】
- 3-5項目を箇条書きで記述
- 日本語対応の状況は必ず含める

* **コスト**: 高性能なモデル（Opus 4.5など）を使用するため、API利用の場合はコストがかさむ可能性がある。

* **人間によるレビュー推奨**: 高度な自律性を持つが、最終的なコードの安全性や意図の確認は人間が行うべきである。
* **日本語対応**: UIやCLIのメッセージは英語ベースであることが多い（日本語での指示は通じる）。

## **13. ベストプラクティス**

<!--
【ガイドライン】
- ツールを効果的に活用するための推奨事項や、避けるべきアンチパターンを記述
- 公式ドキュメントのBest Practicesや、熟練エンジニアの知見を参考にする
-->

* **効果的な活用法 (Modern Practices)**:
  * **検証手段の提供 (Give Claude a way to verify its work)**: テストコード、スクリーンショット比較、期待される出力などを提示し、Claude自身に検証させる。
  * **「調査→計画→実装」のフロー (Explore first, then plan, then code)**: いきなり実装せず、Plan Modeを使ってコードベースの調査と計画立案を行ってから実装に進む。
  * **具体的なコンテキストの提供 (Provide specific context)**: `@`で特定のファイルを参照し、制約条件や既存のパターン（例：「HotDogWidget.phpを参考にして」）を具体的に指示する。
  * **CLAUDE.mdの整備 (Write an effective CLAUDE.md)**: プロジェクト固有のビルドコマンドやスタイルガイドを`CLAUDE.md`に記述し、永続的なコンテキストとして共有する（ただし簡潔に保つ）。
  * **CLIツールの活用 (Use CLI tools)**: `gh` (GitHub CLI) や `aws` などのCLIツールを活用させ、外部サービスとの連携を効率化する。
  * **権限設定とサンドボックス (Configure permissions)**: 頻繁に使うコマンドを許可リストに登録するか、サンドボックス機能を活用して安全かつスムーズに実行させる。
* **陥りやすい罠 (Antipatterns)**:
  * **キッチンシンクセッション (The kitchen sink session)**: 1つのセッションで無関係なタスクを次々と行うとコンテキストが汚染される。タスクごとに`/clear`でリセットすべき。
  * **終わらない修正ループ (Correcting over and over)**: 何度指摘しても修正できない場合は、コンテキストが混乱している可能性が高い。`/clear`して、より明確なプロンプトでやり直す方が早い。
  * **CLAUDE.mdの肥大化 (The over-specified CLAUDE.md)**: ルールが多すぎると無視される原因になる。本当に必要な情報だけに絞り込む。
  * **検証なき信頼 (The trust-then-verify gap)**: エッジケースの考慮漏れがあるため、必ずテストや検証手順を実行させて確認する。「検証できないなら出荷しない」原則を守る。
  * **無制限の調査 (The infinite exploration)**: 範囲を指定せずに「調査して」と指示すると、大量のファイルを読み込んでコンテキストを浪費する。範囲を絞るか、サブエージェントを活用する。

| **Claude Code** | ターミナル/IDE/Webで動作する自律エージェント。 | 環境を選ばず、深いコード理解と自律的なタスク実行が可能。MCPによる拡張性が高い。 | エディタ機能そのものは持たない（IDE拡張として動作）。 | ターミナル中心の開発や、自律的なタスク遂行を重視する場合。 |
| **Cursor** | AIネイティブなコードエディタ。 | エディタ体験全体がAIに最適化されており、UXが非常に優れている。 | エディタを変える必要がある（VS Codeフォーク）。 | エディタ体験ごとAI化したい場合。 |
| **GitHub Copilot** | 業界標準のAIコーディング支援。 | GitHubエコシステムとの統合が強力。多くのIDEで使える。 | 自律的なエージェント機能（複雑な計画立案など）ではClaude Code/Cursorに劣る場合がある。 | 既存のワークフローを維持しつつAI支援を受けたい場合。 |

## **17. 総評**

<!--
【ガイドライン】
- 総合評価、推奨チーム、選択ポイントの3項目を必ず含める
- 客観的で中立的な評価を心がける
-->

* **総合的な評価**:
  Claude Codeは、最新のClaude 4.5モデルの能力を最大限に引き出し、開発者の「手足」となるだけでなく「頭脳」としても機能する強力なエージェントである。ターミナル、IDE、Slackと場所を選ばずに利用できる柔軟性と、MCPによる拡張性は他ツールに対する大きなアドバンテージである。
* **推奨されるチームやプロジェクト**:
  自律的なツールを使いこなせるエンジニアチーム。複雑なコードベースを持ち、調査やリファクタリングに時間を取られているプロジェクト。
* **選択時のポイント**:
  GUIエディタでのシームレスな体験を最優先するならCursor、ターミナルでの操作性やパイプラインへの組み込み、自律的なタスク遂行を重視するならClaude Codeが適している。
