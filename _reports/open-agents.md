# **Open Agents 調査レポート**

## **1. 基本情報**

* **ツール名**: Open Agents
* **ツールの読み方**: オープンエージェンツ
* **開発元**: Vercel Labs
* **公式サイト**: [https://open-agents.dev/](https://open-agents.dev/)
* **関連リンク**:
  * GitHub: [https://github.com/vercel-labs/open-agents](https://github.com/vercel-labs/open-agents)
* **カテゴリ**: AI開発エージェント
* **概要**: Vercel上でクラウドベースのコーディングエージェントを構築・実行するためのオープンソースのリファレンスアプリケーション。エージェントの実行環境とサンドボックスを明確に分離している点が特徴である。

## **2. 目的と主な利用シーン**

* **解決する課題**: ローカルマシンのリソースや環境に依存せず、クラウド上で継続的・自律的にコード生成やタスク実行を行う。
* **想定利用者**: 開発者、AIアプリケーション開発者、インフラエンジニア。
* **利用シーン**:
  * クラウド上での自律的なリポジトリクローンとブランチ作成・コード修正
  * 自動コミットやPull Requestの自律的な生成
  * 複数ステップにわたる長時間のコーディングタスクの実行

## **3. 主要機能**

* **ファイル・シェル操作**: ファイルの読み書き、検索、シェルコマンドの実行など、システムレベルの操作ツールを標準搭載。
* **分離されたサンドボックス**: エージェントの実行環境とサンドボックス環境（Vercel Sandbox）が分離されており、独立して休止・再開が可能。
* **耐久性のあるワークフロー**: VercelのWorkflow SDKを使用し、再起動やマルチステップの操作に耐えうるレジリエントなエージェントループ。
* **自動Git統合**: 成功した実行後に自動でコミット、プッシュ、Pull Requestを作成する機能。
* **マルチモデル対応**: AI SDKおよびAI Gatewayを介した複数AIプロバイダ（モデル）の利用。
* **音声入力機能**: ElevenLabsを利用したオプションの音声入力・文字起こし機能。

## **4. 開始手順・セットアップ**

* **前提条件**:
  * Node.js (Bun推奨)
  * Vercelアカウント
  * GitHubアカウント
* **インストール/導入**:

  ```bash
  git clone https://github.com/vercel-labs/open-agents
  cd open-agents
  bun install
  ```

* **初期設定**:
  * Vercel上の環境変数設定、Better Authのためのシークレット生成。
  * Vercel OAuthアプリおよびGitHub Appの作成と設定。
* **クイックスタート**:
  * ローカル環境で `bun run web` を実行し、開発サーバーを起動する。

## **5. 特徴・強み (Pros)**

* **アーキテクチャの分離**: エージェントとサンドボックスが分離されていることで、安全でスケーラブルな実行環境を提供。VM自体がコントロールプレーンにならない設計。
* **Vercelインフラのフル活用**: AI SDK, AI Gateway, Vercel Sandbox, Workflow SDKなど、本番環境レベルのインフラを利用している。
* **耐久性**: 1回のリクエストライフサイクルに縛られず、長時間の自律タスクを確実に実行できるワークフロー設計。

## **6. 弱み・注意点 (Cons)**

* **初期設定の煩雑さ**: VercelやGitHub AppのOAuth設定、多数の環境変数など、セットアップ項目が多く複雑である。
* **サポート体制**: あくまでリファレンスアプリという位置づけであり、商用SaaSのような手厚いサポートやSLAは期待できない。
* **日本語対応**: 公式ドキュメントやUIは英語ベースであり、情報のキャッチアップには英語の理解が必要。

* **導入事例**: Vercelインフラを活用したAIエージェント構築のベストプラクティスとして、開発者コミュニティで検証・利用が進んでいる。
* **対象業界**: ソフトウェア開発、AIサービス開発企業。

## **9. サポート体制**

* **ドキュメント**: GitHubのREADMEが主要なドキュメントとなっており、詳細なセットアップガイドやアーキテクチャの解説が記載されている。
* **コミュニティ**: GitHubのIssuesやDiscussionsで開発者同士のやり取りが行われている。
* **公式サポート**: オープンソースのリファレンスアプリであるため、Vercelからの公式な個別サポートはない。

## **10. エコシステムと連携**

### **10.1 API・外部サービス連携**

* **API**: AI SDK, AI Gatewayを活用し、様々なLLMプロバイダへの透過的なアクセスを実現している。
* **外部サービス連携**: GitHub（リポジトリの読み書き、PR作成）、ElevenLabs（音声入力）、Vercel（データベース、キャッシュ、ホスティング）。

### **10.2 技術スタックとの相性**

| 技術スタック | 相性 | メリット・推奨理由 | 懸念点・注意点 |
|:---|:---:|:---|:---|
| **Vercel Ecosystem** | ◎ | Vercel SandboxやWorkflow SDKなど、Vercelのインフラをフル活用する設計。 | Vercel以外の環境への移植は困難。 |
| **Next.js** | ◎ | フロントエンド（web UI）はNext.jsで構築されており、親和性が非常に高い。 | 特になし |
| **Bun** | ◯ | パッケージマネージャおよびランタイムとしてBunが推奨されている。 | Node.jsの完全互換に依存する一部パッケージで問題が生じる可能性がある。 |

## **11. セキュリティとコンプライアンス**

* **認証**: Better Authを使用し、VercelおよびGitHubのOAuthを通じた安全な認証を実装している。
* **データ管理**: Neon PostgresやVercel KVを利用し、セッションやスキル情報を管理。サンドボックス環境は独立したVMで実行されるため、ホスト側への影響が遮断される設計。
* **準拠規格**: オープンソースプロジェクトのため、SOC2やISO27001などの公式な認証取得状況は公開されていない。デプロイ先のインフラ（Vercel等）の規格に依存する。

## **12. 操作性 (UI/UX) と学習コスト**

* **UI/UX**: モダンなチャットベースのUIを採用。タスクの進捗状況やエージェントが実行している操作（シェルコマンド、ファイル編集など）をリアルタイムで確認できるストリーミングUIが備わっている。
* **学習コスト**: Web・Workflow・Sandboxという3層構造のアーキテクチャの理解と、Vercelインフラ特有の設定が必要となるため、中程度の学習コストがかかる。

## **13. ベストプラクティス**

* **効果的な活用法 (Modern Practices)**:
  * 本リポジトリをそのまま使うのではなく、フォークして独自のツールや機能をエージェントに追加し、カスタマイズのベースラインとして利用することが推奨される。
* **陥りやすい罠 (Antipatterns)**:
  * エージェントの自律ループを長時間の放置や無制限に実行させること。Workflow SDKによりレジリエントに稼働し続けるため、APIコストが意図せず増大する可能性がある。

## **14. ユーザーの声（レビュー分析）**

* **総合評価**: G2やCapterraでの評価はないが、GitHub上で5,000以上のStarsを獲得しており、開発者の関心が非常に高い。
* **ポジティブな評価**:
  * 「Vercelのインフラを活用した実用的なAIエージェントのアーキテクチャとして非常に参考になる」
  * 「サンドボックスが分離されていることで、安全かつスケーラブルにエージェントを実行できる設計が素晴らしい」
* **ネガティブな評価 / 改善要望**:

* **特徴的なユースケース**:
  * 自社専用のクラウドベース開発ボットや自動PRレビューシステムのベースコードとしての利用。

## **17. 総評**

* **総合的な評価**:
  * Vercelのインフラストラクチャ（AI SDK, AI Gateway, Sandbox, Workflow）を駆使した、実践的かつクラウドネイティブなAIエージェントのリファレンス実装として非常に優れている。特に、エージェント実行環境とサンドボックスを分離するというアーキテクチャは、スケーラビリティとセキュリティの両面において高く評価できる。
* **推奨されるチームやプロジェクト**:
  * すでにVercelエコシステムを活用しており、自社独自のAI開発エージェントや自動化ツールを構築・カスタマイズしたい開発チームやスタートアップに最適である。
* **選択時のポイント**:
  * 完全なSaaSソリューション（Devin等）とは異なり、開発者がフォークして自らの要件に合わせて拡張することが前提となっている。手軽さを求める場合はSaaSを、カスタマイズ性とVercelインフラのフル活用を重視する場合は本ツールを選択するのが良い。
