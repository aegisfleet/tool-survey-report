# **9router 調査レポート**

## **1. 基本情報**

* **ツール名**: 9router
* **ツールの読み方**: 不明
* **開発元**: decolua
* **公式サイト**: [https://9router.com/](https://9router.com/)
* **関連リンク**:
  * GitHub: [https://github.com/decolua/9router](https://github.com/decolua/9router)
* **カテゴリ**: 不明
* **概要**: 9routerは、AIコーディングツールとLLMプロバイダの間に配置される自己ホスト型のAI APIプロキシです。OpenAI互換のエンドポイントを公開し、トークンの圧縮やプロバイダ間の自動ルーティングを行うことで、コスト削減と高可用性を実現します。

## **2. 目的と主な利用シーン**

* **解決する課題**: AIコーディングツールでの大量のトークン消費によるコスト増大、およびAPIの利用制限（クォータ）到達時のダウンタイム。
* **想定利用者**: Claude Code, Cursor, Cline, CopilotなどのAIコーディングツールを利用する開発者。
* **利用シーン**:
  * AIコーディングツールからの出力を圧縮してトークンコストを削減したい場合。
  * 複数のLLMプロバイダ（有料・無料）を組み合わせて、API制限に達した際に自動的に別のプロバイダへ切り替えたい場合。
  * 複数のAPIキーをローカルのダッシュボードで一元管理したい場合。

## **3. 主要機能**

* **RTK Token Saver (トークン圧縮)**: git diff、grep結果、ディレクトリツリーなどの巨大なツール出力データを圧縮し、リクエストごとの入力トークンを20～40%削減します。
* **マルチプロバイダ・ルーティング**: 40以上のLLMプロバイダに対応しており、クォータ制限に達した際、サブスクリプション → 予算 → 無料枠のプロバイダへと優先順位に従って自動的にフォールバックします。
* **ラウンドロビン・スケジューリング**: プロバイダごとに複数のアカウントを登録し、月間の利用制限がリセットされるまでアカウント間でリクエストを均等に振り分け、クォータを最大限に活用します。
* **OpenAI互換API**: `/v1` エンドポイントを提供し、OpenAIプロトコルに対応したツールであれば、設定変更なしにそのまま接続して利用可能です。
* **Caveman Mode**: 出力プロンプトに対して簡潔に回答するよう指示を注入することで、モデルの応答長（出力トークン）を最大65%削減します。
* **Webダッシュボード管理**: プロバイダの認証情報やルーティングのルールを、セルフホストされた管理画面から一元的にコントロール可能です。

## **4. 開始手順・セットアップ**

* **前提条件**:
  * DockerやNode.js（npm）などの実行環境（詳細な必要条件は不明、一般的なOSSとしての想定）。
* **インストール/導入**:

  ```bash
  # GitHubリポジトリの一般的な導入手順の想定
  git clone https://github.com/decolua/9router.git
  cd 9router
  # .envファイルの作成など
  ```

* **初期設定**:
  * Webダッシュボード（デフォルトは通常 `http://localhost:PORT`）にアクセスし、使用するLLMプロバイダのAPIキーやアカウント情報を設定します。
  * 各モデルに対するトークン予算やフォールバックの優先順位を定義します。
* **クイックスタート**:
  * AIコーディングツール（例: Cursor）のAPIエンドポイント設定を、9routerが公開するOpenAI互換エンドポイント（例: `http://localhost:PORT/v1`）に変更します。

## **5. 特徴・強み (Pros)**

* コスト削減効果が極めて高い（RTK Token Saverによる入力トークン削減と、Caveman Modeによる出力トークン削減の組み合わせ）。
* ツール側の設定を変えることなく、シームレスに複数のプロバイダ間でのフェイルオーバーが実現できる。
* セルフホスト型のため、APIキーやプロンプトデータなどの機密情報が外部の管理サーバーに渡らず、自身のコントロール下で安全に運用できる。
* 多数の無料プロバイダ（Free Claude/GPT/Gemini等）へのルーティングをサポートしている。

## **6. 弱み・注意点 (Cons)**

* 自身でVPSやローカル環境へデプロイして運用する（セルフホスト）手間がかかる。
* 間に入るプロキシサーバーとして動作するため、ネットワークの構成やプロバイダのレスポンス状況によってはわずかな遅延（レイテンシ）が生じる可能性がある。
* 日本語へのローカライズ状況は不明。

## **13. ベストプラクティス**

* **効果的な活用法 (Modern Practices)**:
  * サブスクリプション枠を使い切った後、自動的に低コストモデルや無料枠のプロバイダに切り替わるようにフォールバックのルールを設定することで、ダウンタイムなしに開発を継続する。
  * RTK Token SaverとCaveman Modeを併用して、入力と出力の両面からトークン消費を最小限に抑える。
* **陥りやすい罠 (Antipatterns)**:
  * 無料プロバイダのみに依存したルーティング設定にすると、各プロバイダの制限やAPI仕様の変更により突然利用できなくなるリスクがある。

## **17. 総評**

* **総合的な評価**:
  * 9routerは、AIコーディングツールを日常的に酷使する開発者にとって、コスト削減の強力な味方となるツールです。RTK Token Saverなどの独自のトークン圧縮機能と、複数プロバイダ間の賢いフォールバック機能により、API制限のストレスから解放されます。完全なセルフホスト型OSSであるため、プライバシーとコントロールを維持できる点も優れています。
* **推奨されるチームやプロジェクト**:
  * AIコーディング支援ツール（Cursor, Cline, Copilot等）の利用頻度が非常に高く、APIコストの増大に悩んでいる個人開発者や小規模チーム。
* **選択時のポイント**:
  * 法人としての公式サポートやSLAが不要であり、自身の環境にデプロイして運用するリテラシーがある場合は、コストパフォーマンスの観点から最有力な選択肢となります。
