# **AFFiNE 調査レポート**

## **1. 基本情報**

* **ツール名**: AFFiNE
* **ツールの読み方**: アファイン
* **開発元**: Toeverything
* **公式サイト**: [https://affine.pro/](https://affine.pro/)
* **関連リンク**:
  * GitHub: [https://github.com/toeverything/AFFiNE](https://github.com/toeverything/AFFiNE)
  * ドキュメント: [https://docs.affine.pro/](https://docs.affine.pro/)
  * レビューサイト: [Product Hunt](https://www.producthunt.com/products/affine-2)
* **カテゴリ**: ナレッジ管理
* **概要**: AFFiNEは、ドキュメント作成、ホワイトボード、データベース機能を一つに統合した、ローカルファーストでプライバシー重視のオープンソースナレッジ管理ツールです。NotionとMiroの代替として、ブロックベースのエディタとエッジレスなキャンバスをシームレスに行き来しながら情報を整理・視覚化できます。

## **2. 目的と主な利用シーン**

* **解決する課題**: 複数のツール（ドキュメントツール、ホワイトボードツール、タスク管理ツール）を切り替えることによるコンテキストスイッチの負担や、クラウド依存によるデータの所有権・プライバシーの懸念を解決します。
* **想定利用者**: 開発者、デザイナー、プロダクトマネージャー、学生、およびプライバシーを重視するチームや企業。
* **利用シーン**:
  * チームでのプロジェクト計画、タスクの整理、およびマインドマップ等を用いた視覚的なブレインストーミング。
  * ローカル環境でセキュアに個人やチームのナレッジベース（Wiki）を構築・運用する。
  * AIパートナー（AFFiNE AI）を活用したレポート作成、スライドの生成、ドキュメントの要約。

## **3. 主要機能**

* **統合されたワークスペース (Docs & Edgeless)**: リッチテキストエディタ（Docs）と無限に広がるホワイトボード（Edgeless）をボタン一つでシームレスに切り替えることができます。
* **ブロックベースのエディタ**: テキスト、画像、データベース、図形などをすべて「ブロック」として扱い、自由に配置・再構成が可能です。
* **ローカルファーストとリアルタイムコラボレーション**: データはまずローカルに保存され（オフラインファースト）、ネットワーク接続時にクラウド（AFFiNE Cloud）を通じてリアルタイム同期・コラボレーションが可能です。
* **AFFiNE AI**: ドキュメントの要約、マインドマップの自動生成、プレゼンテーション用スライドの作成などを支援するマルチモーダルなAIコパイロット機能。
* **データベース・テーブルビュー**: カンバン、テーブルなど複数のビューを持つデータベースで、タスクや情報を構造化して管理できます。
* **セルフホスト対応**: Dockerなどを用いて自社サーバーにデプロイし、完全にデータをコントロールすることが可能です。

## **4. 開始手順・セットアップ**

* **前提条件**:
  * Webブラウザ、またはデスクトップアプリ（Windows, macOS, Linux）のインストール。
  * クラウド同期機能を利用する場合はアカウント登録が必要。ローカル利用のみならアカウント不要。
  * セルフホストする場合はDocker環境が必要。
* **インストール/導入**:
  * デスクトップアプリは公式サイトからダウンロードするか、GitHub Releasesから取得します。
* **セルフホスト (Docker)**:
  ```bash
  # Dockerを用いたセルフホストの基本コマンド例（公式ドキュメント参照）
  docker run -d --name affine -p 3020:3020 ghcr.io/toeverything/affine:latest
  ```
* **クイックスタート**: アプリを起動（またはWeb版にアクセス）するとローカルワークスペースが即座に作成され、すぐにドキュメントやホワイトボードの作成を開始できます。

## **5. 特徴・強み (Pros)**

* **Docとホワイトボードの完全統合**: テキスト文書とビジュアルキャンバスを同じデータソースで表現し、シームレスに切り替えられる点が、他のツール（Notion等）にはない最大の強みです。
* **ローカルファースト・プライバシー重視**: データはオフラインでも利用可能であり、ベンダーロックインを避けることができます。
* **オープンソース**: 透明性が高く、活発なコミュニティによって頻繁にアップデート（Canaryリリースなど）が行われています。
* **オールインワン**: 複数のツールを組み合わせる必要がなくなり、ツールチェーンの複雑さを軽減できます。

## **6. 弱み・注意点 (Cons)**

* **機能の成熟度**: 発展途上のツールであり、ユーザーレビューによるとPDFエクスポートの制限、高度な検索機能、置換機能、バックリンクやタグシステムの充実度など、Notion等と比較して不足している部分があります。
* **AI機能のUI/UX**: 一部のレビューにおいて、AI機能の呼び出しやワークフローが競合ツール（Notion AIなど）に比べて直感的でなく、手数がかかると指摘されています。
* **パフォーマンス**: 大量のブロックや複雑なキャンバスを扱う際に、ラグや小さなバグが発生することが報告されています。
* **日本語対応**: コミュニティ等での日本語情報は増加していますが、国内特化のSaaSに比べると日本語サポートの面でハードルがある場合があります。

## **10. エコシステムと連携**

### **10.1 API・外部サービス連携**

* **API**: プロダクトはオープンソースとして公開されており、バックエンド（OctoBase）やエディタ（BlockSuite）のアーキテクチャが公開されていますが、一般的なSaaSのようなREST APIのサードパーティ向け連携機能（ZapierやMake等）は現在発展途上またはコミュニティプラグインに依存しています。
* **外部サービス連携**: Webクリッパー（Chrome拡張）が提供されており、ブラウザからの情報収集が可能です。

### **10.2 技術スタックとの相性**

| 技術スタック | 相性 | メリット・推奨理由 | 懸念点・注意点 |
|:---|:---:|:---|:---|
| **TypeScript / React** | ◎ | フロントエンド（エディタ含む）がReactとTypeScriptで構築されており、OSSコントリビューションや拡張が容易。 | 特になし |
| **Rust** | ◎ | バックエンドのCRDTエンジン（y-octo, OctoBase）にRustが採用され、高性能。 | Rustの知見がないとバックエンドの深いカスタマイズは困難。 |
| **Docker** | ◎ | 公式にDockerイメージが提供されており、セルフホストが容易。 | インフラの保守管理は自前で行う必要がある。 |

## **11. セキュリティとコンプライアンス**

* **認証**: クラウド版（AFFiNE Cloud）はアカウントベースの認証。将来のエンタープライズ版（EE）ではSSOや高度な監査機能が予定されています。
* **データ管理**: **ローカルファースト**アーキテクチャにより、データはデフォルトでユーザーのローカルディスクに保存されます（プライバシー重視）。クラウド同期を利用する場合のみサーバーにデータが送信されます。
* **準拠規格**: セルフホスト版を利用することで、社内のセキュリティポリシー（データ保管場所の国内限定など）に完全に準拠させることが可能です。

## **12. 操作性 (UI/UX) と学習コスト**

* **UI/UX**: 非常にクリーンでモダンなインターフェース。ボタン一つでドキュメントビューとホワイトボード（Edgeless）ビューを切り替えられる革新的なUXを提供しています。
* **学習コスト**: Notionなどのブロックエディタに慣れているユーザーであれば直感的に操作できますが、多機能（Doc、ホワイトボード、データベース、AI）であるため、機能を完全に使いこなすための学習コストはやや高めです。

## **13. ベストプラクティス**

* **効果的な活用法 (Modern Practices)**:
  * 会議や講義のテキストメモをDocsモードで取り、その後Edgelessモードに切り替えてマインドマップやフローチャートとして視覚的に整理する。
  * プライバシーが極めて重要なプロジェクト（機密情報を含む開発など）でセルフホスト版を利用し、ローカルファーストの強みを活かす。
* **陥りやすい罠 (Antipatterns)**:
  * 他の専用ツール（本格的なプロジェクト管理ツールなど）と完全に同じ機能を期待すること。Notionと比較して一部の細かな機能（高度な検索、置換など）が不足している場合があるため、チームの要件と合致するか事前に検証が必要です。

## **17. 総評**

* **総合的な評価**:
  AFFiNEは、Notionの強力なドキュメント・データベース機能とMiroの柔軟なホワイトボード機能を見事に融合させた野心的なプロジェクトです。特にローカルファーストを原則とし、データを自身のコントロール下に置けるオープンソースアーキテクチャは、クラウドSaaSへのロックインを懸念するユーザーにとって非常に魅力的です。発展途上ゆえの細かな機能不足はありますが、急速なアップデートにより改善が進んでいます。
* **推奨されるチームやプロジェクト**:
  * プライバシーやデータセキュリティの要件が厳しい開発チームや企業。
  * テキストによる仕様記述と、アーキテクチャ図などのビジュアル表現を頻繁に行き来するエンジニアやデザイナーのチーム。
  * オープンソースを好み、自社インフラにナレッジベースをセルフホストしたい組織。
* **選択時のポイント**:
  * 既存のNotionなどのワークフローから移行する場合、検索やエクスポートなどの一部機能の差異を事前に確認することが重要です。
  * テキスト管理だけでなく、ホワイトボードでの視覚的な整理を日常的に行うかどうかが、導入のメリットを最大化する鍵となります。
