# **Cursor 調査レポート**

## **1. 基本情報**

* **ツール名**: Cursor
* **開発元**: Anysphere, Inc.
* **公式サイト**: [https://cursor.com/](https://cursor.com/)
* **カテゴリ**: AIコードエディタ
* **概要**: Cursorは、AIを活用したコーディング支援機能が組み込まれた最新のコードエディタです。Visual Studio Code (VS Code) をベースに開発されており、使い慣れたインターフェースに強力なAI機能を統合しています。

## **2. 目的と主な利用シーン**

* **目的**: 開発者の生産性向上、コーディングの高速化、品質向上
* **主な利用シーン**:
    * コードの自動生成・補完
    * 複雑なコードの理解・リファクタリング
    * バグの特定と修正
    * ドキュメント作成の効率化
    * 新しい技術やライブラリの学習

## **3. 主要機能**

* **AIエージェント機能**:
    * **Multi-Agent**: 複数のAIエージェントを並行して実行し、それぞれが独立した環境でコード変更案を生成。
    * **Composerモデル**: エージェント機能に特化して開発された高速・高機能な独自モデル。
    * **Plan Mode**: 複雑なタスクに対して、AIがまず実行計画を立案し、承認後にコーディングを開始。
    * **Voice Mode**: 音声入力でAIエージェントを操作可能。
    * **Hooks**: カスタムスクリプトにより、エージェントの動作を監視、制御、拡張可能。
* **コーディング支援**:
    * **AIチャット**: エディタ内でAIと対話し、コードの生成、修正、デバッグを依頼可能。コードベース全体を認識した上での回答が得られる。
    * **インライン編集**: `Cmd+K`（または`Ctrl+K`）で、自然言語の指示により既存のコードを直接編集。
    * **インテリジェントな自動補完**: `Tab`キーで、AIが文脈を読んで次のコードを高速かつ正確に予測・補完。
* **開発ワークフロー連携**:
    * **Browser機能**: エージェントがブラウザを操作し、Web上の情報を収集したり、UIのデバッグを行ったりできる。
    * **Sandboxed Terminals**: AIが実行するコマンドをサンドボックス化し、安全な環境で実行。
    * **各種ツール連携**: GitHub、Slack、Linearと統合し、PRレビュー、チーム内コミュニケーション、タスク管理をシームレスに行う。
* **チーム開発支援**:
    * **Team Commands**: チーム内で共通のカスタムコマンドやプロンプトを定義・共有可能。
    * **Bugbot**: GitHubと連携し、PRの自動レビューや修正提案を行う。

## **4. 特徴・強み (Pros)**

* **VS Codeベースの優れたUX**: 既存のVS Codeユーザーが、使い慣れたキーバインドや拡張機能を維持しつつ、スムーズに移行可能。
* **高度なAIエージェント機能**: 複数のエージェントを並行稼働させたり、実行計画を立てさせたりと、単なるコード補完に留まらない自律的な開発支援を実現。
* **深いコードベース理解**: プロジェクト全体の文脈をインデックス化し、コードの依存関係や構造を理解した上で、的確なコード生成・修正が可能。
* **柔軟なモデル選択**: OpenAI、Anthropic、Gemini、xAIなど、主要な最新AIモデルから最適なものを選択して利用可能。
* **強力なエコシステム連携**: GitHub、Slack、Linearといった開発ツールと深く連携し、開発ワークフロー全体を効率化。

## **5. 弱み・注意点 (Cons)**

* **セキュリティリスク**: AIによるコード実行には、コマンドインジェクションやデータ漏洩のリスクが伴う
* **プライバシー**: プライバシーモードを有効にしない限り、コードや対話履歴がモデルの学習に使用される可能性がある
* **パフォーマンス**: AI機能の使用により、エディタの動作が遅くなる場合がある
* **日本語対応**: UIは日本語に対応しているが、AIの日本語理解度は英語に劣る場合がある
* **Vimキーバインドのサポートが不完全**: 一部のVimキーバインドが動作しない場合がある

## **9. 連携機能 (API・インテグレーション)**

* **GitHub**:
    * **PRレビュー**: `Bugbot`がPull Requestを自動でレビューし、修正案を提示する。
    * **PR概要生成**: PRを作成すると、変更内容のサマリーを自動で生成・更新する。
    * **Background Agents**: PR内で`@Cursor`とメンションすることで、エージェントを呼び出して修正タスクを依頼できる。
* **Slack**: チームのSlackワークスペースと連携し、エディタ外からCursorの機能を利用できる。
* **Linear**: issueから直接Cursorのエージェントを起動し、タスクを委任できる。
* **MCP (Model Context Protocol)**: 対応サーバーと連携し、ファイルやDBスキーマなどの外部コンテキストをAIモデルに提供できる。

## **10. セキュリティとコンプライアンス**

* **認証**: SAML/OIDC SSO (Teamsプラン以上)
* **データ管理**:
    * **プライバシーモード**: チーム管理者が組織全体でプライバシーモードを強制でき、コードや会話履歴がモデル学習に使用されるのを防ぐ。
    * **Sandboxed Terminals**: AIによるコマンド実行をサンドボックス化し、ワークスペースへの読み書きアクセスのみを許可。インターネットアクセスはデフォルトで無効。
* **準拠規格**: SOC 2 Certified

## **13. 直近半年のアップデート情報**

* **2025-10-29 (v2.0)**:
    * **Multi-Agents**: 複数のAIエージェントを並列実行する機能を追加。
    * **Composer**: エージェント機能に特化した独自のコーディングモデルを導入。
    * **Browser (GA)**: エージェントがブラウザを操作する機能が正式リリース。
    * **Sandboxed Terminals (GA)**: AIによるコマンド実行のサンドボックス機能がmacOSで正式リリース。
    * **Voice Mode**: 音声によるエージェント操作機能を追加。
* **2025-09-29 (v1.7)**:
    * **Browser Controls (Beta)**: エージェントがスクリーンショット撮影やUIデバッグを行うブラウザ操作機能を追加。
    * **Plan Mode**: AIが複雑なタスクの実行計画を立案する機能を追加。
    * **Hooks (Beta)**: カスタムスクリプトでエージェントの動作を制御・拡張する機能を追加。
* **2025-09-12 (v1.6)**:
    * **Custom slash commands**: チームで再利用可能なカスタムプロンプトを作成・共有する機能を追加。
    * **Summarization triggers**: 会話履歴が長くなった場合に手動で要約をトリガーする機能を追加。
    * **MCP Resources support**: MCPプロトコル経由で外部リソースをコンテキストとして利用可能に。
* **2025-08-21 (v1.5)**:
    * **Linear integration**: Linearのissueから直接エージェントを起動できる連携機能を追加。
    * **OS notifications**: エージェントの実行完了時などにOSネイティブの通知を受け取れる機能を追加。
* **2025-08-06 (v1.4)**:
    * **GitHub support for Background Agents**: GitHubのPRから直接エージェントを呼び出す機能を追加。
    * **Improved Agent tools**: ファイル読み込み、ディレクトリ探索、grepなどのエージェント用ツールを大幅に改善。

## **15. 総評**

Cursorは、VS Codeの優れたUXと、強力なAI機能を高度に統合した次世代のコードエディタです。単なるコード補完ツールに留まらず、Multi-AgentやPlan Modeといった高度なAIエージェント機能を備え、自律的なタスク実行までを視野に入れています。プロジェクト全体の文脈を深く理解した上でのコード生成・修正、そしてGitHubやLinearといった開発ツールとのシームレスな連携は、開発ワークフローそのものを変革するポテンシャルを秘めています。
無料プランから利用可能で、多くの著名な開発者や企業に支持されていることからも、その実用性の高さがうかがえます。AIによる開発支援の最前線を体験したいすべての開発者にとって、試す価値のある非常に有力な選択肢です。ただし、AIによるコード実行にはセキュリティ等のリスクも伴うため、サンドボックス機能やプライバシーモードの活用を推奨します。
