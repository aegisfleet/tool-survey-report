## 1. 基本情報

- **名称**: HSP3 (Hot Soup Processor)
- **読み方**: エイチエスピースリー / ホットスーププロセッサ
- **カテゴリ**: プログラミング言語 / 統合開発環境 (IDE)
- **開発元**: ONION software (おにたま)
- **公式サイト**: [https://hsp.tv/](https://hsp.tv/)
- **ライセンス**: BSDライセンス (OpenHSPとしてコア部分はオープンソース)
- **対応OS**: Windows, Linux (Raspberry Pi), Android, iOS, Web (HSP3Dish経由)
- **最新バージョン**: 3.7 (2025-09-05リリース)

### 概要
HSP3は、1997年から開発が続けられている日本製のプログラミング言語および開発環境です。「誰でも手軽にWindowsアプリやゲームを作れる」ことをコンセプトに、BASICに似た簡単な文法を採用しています。
初心者向けの学習用言語としてだけでなく、実用的なツールやインディーゲームの開発にも長年利用されています。

### 主な特徴
- **超軽量・インストール不要**: 解凍するだけで開発環境が整う手軽さ。
- **強力なマルチメディア機能**: 画像表示、音楽再生、3D描画(HGIMG4)を標準サポート。
- **マルチプラットフォーム**: HSP3Dishランタイムを使用することで、Android/iOSアプリやブラウザゲーム(WebGL)の作成が可能。
- **コンパイル済み実行ファイル作成**: 単体で動作するEXEファイルを簡単に作成・配布可能。

## 2. 機能調査

### 主要機能リスト

1. **GUIアプリケーション開発**
   - ボタン、入力ボックス、リストボックスなどのWindows標準コントロールを数行のコードで配置可能。
   - `button "押してね", *label` のような直感的な記述。

2. **2Dゲーム開発機能**
   - 高速なスプライト描画、衝突判定、マップ表示などをサポート。
   - DirectX (OpenGL) をラップしており、ハードウェアアクセラレーションを利用可能。

3. **3Dゲーム開発機能 (HGIMG4)**
   - 3Dモデル(.gpb)の読み込み、アニメーション、物理演算(Bullet Physics)をサポート。
   - プログラミング初心者でも3D空間を扱えるように設計されている。

4. **マルチプラットフォーム出力 (HSP3Dish)**
   - 作成したプログラムをAndroid、iOS、WebGL(HTML5)、Linux向けに変換・出力する機能。
   - ワンソースで複数のプラットフォームに対応可能。

5. **COM/DLL連携**
   - Windows APIや外部DLLの呼び出し、COMコンポーネントの利用が可能。
   - Excel操作や高度なシステム制御など、スクリプト言語の枠を超えた拡張性を持つ。

6. **HSPTV!ブラウザ / 素材**
   - ユーザー作品を共有・公開できるプラットフォームと連携。
   - 著作権フリーの画像・音楽素材が標準で同梱されており、すぐにゲーム作りを始められる。

## 4. 開始手順・セットアップ調査

### 前提条件
- **OS**: Windows 10/11 (推奨)
- **ランタイム**: 特になし (同梱されているため)

### インストール手順
1. 公式サイトの[ダウンロードページ](https://hsp.tv/idman/download.html)にアクセス。
2. 「HSP3.7 フルセット」のインストーラー版(推奨)またはzip版をダウンロード。
3. インストーラーを実行して指示に従うか、zipを任意のフォルダに解凍する。

### Hello World
エディタ(HSED3.exe)を起動し、以下のコードを入力してF5キーを押す。

```hsp
mes "Hello World!"
button "終了", *exit
stop

*exit
    end
```

実行するとウィンドウが表示され、"Hello World!"という文字と終了ボタンが表示されます。

## 6. 競合分析

### 機能比較表 (星取表)

| 機能 | HSP3 | Python (Pygame) | Unity | Scratch |
| :--- | :---: | :---: | :---: | :---: |
| **日本語対応** | ◎ | △ (英語ベース) | △ (UIは英語/日本語化可) | ◎ |
| **環境構築の手軽さ** | ◎ | ◯ | △ (重い) | ◎ (ブラウザのみ) |
| **EXE化の容易さ** | ◎ | △ (PyInstaller等が必要) | ◯ (ビルド時間かかる) | × (Webメイン) |
| **3D機能** | ◯ (HGIMG4) | △ | ◎ | △ |
| **スマホ/Web出力** | ◯ (Dish) | △ | ◎ | △ |
| **学習難易度** | 低 | 中 | 高 | 極低 |

- **凡例**:
  - ◎: 非常に優れている / 完全対応
  - ◯: 標準的 / 対応
  - △: 少し手間がかかる / 限定的
  - ×: 非対応 / 困難

### 詳細比較

- **vs Python**: Pythonはライブラリが豊富でAIやデータ分析に強いですが、GUIアプリやゲームを配布可能な形式(EXE)にするには手間がかかります。HSP3は「GUIアプリを作って配る」という点において圧倒的に手軽です。
- **vs Scratch**: Scratchはブロックを組み合わせるビジュアルプログラミングで、タイピングが不要です。HSP3はコードを書くテキストプログラミングへのステップアップとして適しています。
- **vs Unity**: Unityはプロフェッショナルな3Dゲーム開発の標準ですが、機能が多すぎて初心者には学習コストが高いです。HSP3は「とりあえず動くものを作る」体験を最速で得られます。

## 9. エコシステムと連携調査

### 外部連携
- **DLL/COM連携**: Windowsの機能をフルに活用可能。Win32 APIを直接叩くことも容易。
- **プラグインシステム**: `.hpi` や `.dll` 形式で機能を拡張可能。物理演算、通信、高度な画像処理などのプラグインが多数公開されている。

### 技術スタック相性表

| 技術 | 相性 | 説明 |
| :--- | :---: | :--- |
| **VS Code** | ◎ | 拡張機能「language-hsp3」により、シンタックスハイライトや補完が可能。 |
| **Git** | ◯ | テキストベースのソースコードなので管理可能。ただし標準エディタに連携機能はない。 |
| **Excel (COM)** | ◎ | COMオートメーションを利用してExcel帳票の自動作成などが手軽に行える。 |
| **Arduino** | ◯ | シリアル通信機能を用いて、Arduino等のマイコンと連携可能。 |

## 10. ベストプラクティス調査

- **モジュール化**: `module` ～ `global` 命令を使用して、変数名の衝突を防ぎ、再利用可能なコードを作成する。
- **HSP3Dishの活用**: 最初からHSP3Dishの制約(描画命令の違いなど)を意識して作れば、後からスマホやWebに移植しやすくなる。
- **外部エディタの利用**: 規模が大きくなってきたら、標準のHSEDではなくVS Codeなどの高機能エディタを利用することで開発効率が上がる。
- **マクロの活用**: `#define` や `#include` を活用して定数管理やファイル分割を行う。
