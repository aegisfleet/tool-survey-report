# **Bito 調査レポート**

## **1. 基本情報**

* **ツール名**: Bito
* **ツールの読み方**: ビト
* **開発元**: Bito Inc.
* **公式サイト**: [https://bito.ai/](https://bito.ai/)
* **関連リンク**:
  * ドキュメント: [https://docs.bito.ai/](https://docs.bito.ai/)
  * レビューサイト: [G2](https://www.g2.com/products/bito/reviews) | [VS Code Marketplace](https://marketplace.visualstudio.com/items?itemName=Bito.Bito)
* **カテゴリ**: AIコーディング支援
* **概要**: Bitoは、コードベース全体を動的なナレッジグラフとしてインデックス化する「AI Architect」を中核とし、IDEやCLI、Gitワークフロー上で深く文脈を理解したAIコードレビューやコーディング支援を提供するプラットフォーム。

## **2. 目的と主な利用シーン**

* **解決する課題**: 単一ファイルしか理解できないAIによる不正確なコード生成や、レビューにおける見落とし、システム全体の依存関係を無視したバグの混入を防ぐこと。
* **想定利用者**: ソフトウェアエンジニア、コードレビュアー、開発チーム全体。
* **利用シーン**:
  * プルリクエスト作成前のローカルIDEでの事前レビューや、作成時の自動コードレビュー。
  * システムの依存関係やAPI、アーキテクチャについてAIに質問しながらのコーディング。
  * 新規参画メンバーの迅速なオンボーディングのためのシステム理解支援。

## **3. 主要機能**

* **AI Architect**: コードベース（リポジトリ、モジュール、APIなど）の動的なナレッジグラフを構築し、AIエージェントに深いシステム文脈を提供する基盤機能。
* **AI Code Review Agent (Git連携)**: GitHub、GitLab、Bitbucketのプルリクエストに対して、シニアエンジニアレベルの自動コードレビュー、バグ・セキュリティリスクの指摘を行う機能。
* **AI Code Review Agent (IDE連携)**: VS CodeやJetBrainsなどのIDE内で、コードを書きながら行単位のレビューや1クリック修正提案を受けられる機能。
* **AI Code Review in CLI**: ターミナルから直接AIコードレビューを実行し、CI/CDパイプラインにも組み込める機能。
* **プロジェクト固有ルール (Guideline) の適用**: `.cursor/rules/*.mdc` や `CLAUDE.md` などのガイドラインファイルを読み込み、プロジェクトのコーディング規約に沿ったレビューを行う機能。
* **Jira / Confluence連携**: PRの要件がJiraチケットやConfluenceドキュメントと合致しているかをAIがクロスチェックする機能。

## **4. 開始手順・セットアップ**

* **前提条件**:
  * Bitoアカウントの作成
  * IDE連携の場合は対応するIDE（VS Code, JetBrains系など）のインストール
* **インストール/導入**:
  * IDE拡張機能の場合：VS Code MarketplaceやJetBrains Pluginsから「Bito」を検索してインストール。
  * Git連携の場合：Bito Cloudからリポジトリを連携設定する（Test Modeを使用すれば個人アクセストークンのみですぐに試用可能）。
* **初期設定**:
  * Bitoアカウントでログインし、ワークスペースを作成。
* **クイックスタート**:
  * Git連携ではTest Modeを利用し、任意のPRのURLを入力するだけで最初のレビュー結果を確認できる。

## **5. 特徴・強み (Pros)**

* **コードベース全体の深い理解 (AI Architect)**: 従来のEmbedding（検索ベース）とは異なり、ナレッジグラフを構築するため、依存関係やシステム全体のアーキテクチャを理解した上での高精度な回答やレビューが可能。
* **幅広いインテグレーション**: Gitプロバイダー（GitHub, GitLab, Bitbucket）だけでなく、多様なIDE（VS Code, JetBrains）やCLI、さらにはMCPを通じて他のAIエージェント（Cursor, Windsurf, Claude Codeなど）とも連携可能。
* **既存の規約ファイルを活用**: CursorやWindsurfなどで使われるルールファイル（`.mdc` など）をそのまま解釈し、レビューに反映できる。

## **6. 弱み・注意点 (Cons)**

* 完全なコードベース理解（AI Architect）を活用するためには、初期のインデックス構築やリポジトリ連携の手間がかかる。
* クラウド版で複数リポジトリをまたぐ大規模な連携を行う場合、セキュリティ要件（オンプレミス展開など）によってはEnterpriseプランの契約が必要となる。
* G2などのレビューサイトでは、まだ他の老舗ツールと比べてレビュー数が少なめである。

## **8. 導入実績・事例**

* **導入企業**: Bito公式によると、多数の開発チームで利用されており、PRのマージ速度が大幅に向上したことが報告されている。
* **導入事例**: エンジニアのレビュー時間を週に数時間削減し、オンボーディング期間を数週間から数日に短縮した等の効果が報告されている。AIがPRフィードバックの約87%を提供することで、ROIが高いとされている。
* **対象業界**: 複雑なコードベースを持つソフトウェア開発企業、スタートアップからエンタープライズまで。

## **10. エコシステムと連携**

### **10.1 API・外部サービス連携**

* **API**: MCP (Model Context Protocol) サーバーとして動作し、外部のAIエージェントにコンテキストを提供可能。
* **外部サービス連携**:
  * **Git**: GitHub, GitLab, Bitbucket（クラウド版・セルフホスト版）
  * **プロジェクト管理**: Jira, Confluence

### **10.2 技術スタックとの相性**

| 技術スタック | 相性 | メリット・推奨理由 | 懸念点・注意点 |
|:---|:---:|:---|:---|
| **IDE (VS Code / JetBrains)** | ◎ | 専用の公式プラグインがあり、リアルタイムなコードレビューが可能 | 特になし |
| **AIエディタ (Cursor / Windsurf)** | ◎ | MCP連携により、Bitoのナレッジグラフを他エディタから利用可能 | MCPの設定が別途必要 |
| **CI/CD** | ◯ | CLIツールを用いたパイプライン組み込みが可能 | Professionalプラン以上で本格活用を推奨 |

## **11. セキュリティとコンプライアンス**

* **認証**: OAuth（GitHub/GitLab等）、エンタープライズ向けにSSO対応。
* **データ管理**: Bitoはユーザーのコードを保存せず、LLMの学習データとしても利用しないと明記している。
* **準拠規格**: SOC2 Type II 認定を取得しており、エンタープライズ基準のセキュリティを備えている。

## **12. 操作性 (UI/UX) と学習コスト**

* **UI/UX**: IDEのサイドバーにシームレスに統合されており、チャット形式での質問や、PR上のコメントとして直接フィードバックを受け取れるため直感的。
* **学習コスト**: プラグインのインストールやTest Modeを使った導入は非常に簡単。一方で、リポジトリ単位の `.bito.yaml` 設定やカスタムルールの記述にはドキュメントの参照が必要。

## **13. ベストプラクティス**

* **効果的な活用法 (Modern Practices)**:
  * プロジェクト内に `.cursor/rules/*.mdc` や `CLAUDE.md` といったガイドラインファイルを配置し、BitoのAIレビューにプロジェクト独自の規約を強制させる。
  * JiraチケットIDをPRのタイトルや説明に含め、要件が満たされているかの自動クロスチェック機能を利用する。
* **陥りやすい罠 (Antipatterns)**:
  * 一部のリポジトリのみ連携してしまい、AI Architectがシステム全体の依存関係（他マイクロサービスなど）を把握できず、結果として浅いレビューになってしまうこと。

## **17. 総評**

* **総合的な評価**:
  Bitoは、単純なコード補完やファイル単位のコンテキストにとどまらず、「AI Architect」によるナレッジグラフを通じてコードベース全体のアーキテクチャや依存関係を深く理解できる点が最大の魅力です。IDEでのリアルタイムな支援と、PR上での詳細なコードレビューの両方を強力にサポートしており、開発効率とコード品質の向上に大きく寄与します。
* **推奨されるチームやプロジェクト**:
  * 複数のマイクロサービスやリポジトリにまたがる複雑なシステムを開発しているチーム。
  * PRのレビューに多くの時間を割かれている、あるいは属人的なレビューでバグの見落としが発生しているチーム。
* **選択時のポイント**:
  Cursorのようなエディタ特化型ツールと競合する側面もありますが、BitoはMCP（Model Context Protocol）を通じてCursorやWindsurfなどの他エディタにもコンテキストを提供できるため、単体のエディタとしてではなく、チームの「コードベース知能」として組み合わせる使い方も非常に有効な選択肢となります。
