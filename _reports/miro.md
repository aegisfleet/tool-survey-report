# **Miro 調査レポート**

## **1. 基本情報**

* **ツール名**: Miro
* **ツールの読み方**: ミロ
* **開発元**: Miro
* **公式サイト**: [https://miro.com/](https://miro.com/)
* **関連リンク**:
  * ドキュメント: [https://help.miro.com/hc/en-us](https://help.miro.com/hc/en-us)
  * レビューサイト: [G2](https://www.g2.com/products/miro/reviews) | [Capterra](https://www.capterra.com/p/162394/Miro/) | [ITreview](https://www.itreview.jp/products/miro/reviews)
* **カテゴリ**: デザインツール
* **概要**: Miroは、チームが場所を問わずアイデアを共有し、協力してプロジェクトを進めるためのオンラインホワイトボードプラットフォーム（イノベーションワークスペース）。ブレインストーミング、図表作成、アジャイル開発の管理、プロトタイピングなど、多岐にわたる用途で活用されている。

## **2. 目的と主な利用シーン**

* **解決する課題**:
  * リモートワークやハイブリッドワークにおけるコミュニケーションの質とスピードの向上
  * アイデア出しから実行計画までのプロセスの分断解消
  * 複雑なシステムやワークフローの可視化
* **想定利用者**:
  * プロダクトマネージャー、プロジェクトマネージャー
  * ソフトウェアエンジニア、システムアーキテクト
  * UX/UIデザイナー、マーケター
* **利用シーン**:
  * **アジャイル開発**: スプリントプランニング、レトロスペクティブ、カンバンボードでのタスク管理
  * **製品設計**: ユーザーフロー図、ワイヤーフレーム、プロトタイプの作成
  * **ワークショップ**: オンラインでの付箋を使ったブレインストーミングや投票
  * **システム設計**: AWS/Azure構成図やUML図、ER図の作成

## **3. 主要機能**

* **無限キャンバス**: 広大なスペースにテキスト、画像、ドキュメント、付箋などを自由に配置可能。
* **Miro AI**: 自然言語からのマインドマップ生成、付箋の自動要約・分類、コードからの図生成などを支援。
* **高度な作図ツール**: UML、BPMN、ER図、クラウドアーキテクチャ図などを作成するための専門的なシェイプパック。
* **インタラクティブ機能**: タイマー、投票、ビデオ通話、BGM、リアクションなど、会議を活性化する機能群。
* **Miro Prototypes**: 静的なデザインをインタラクティブなプロトタイプに変換し、画面遷移や動作を確認可能。
* **トークトラック (Talktrack)**: ボード上の操作を録画して共有でき、非同期での説明やフィードバックに利用可能。
* **開発者向けプラットフォーム**: REST APIやWeb SDKを利用して、独自のアプリや連携機能を開発可能。

## **4. 開始手順・セットアップ**

* **前提条件**:
  * Webブラウザ（Chrome, Edge, Firefox, Safariなど）またはデスクトップアプリ
  * Miroアカウント（Google, Slack, MicrosoftアカウントなどでのSSOも可能）
* **インストール/導入**:
  * ブラウザ版はインストール不要。
  * デスクトップ版は公式サイトからダウンロード可能（Windows/Mac）。
* **初期設定**:
  * アカウント作成後、チームを作成し、メンバーを招待。
  * 必要に応じてSlackやJiraなどの連携アプリをインストール。
* **クイックスタート**:
  1. 「New board」をクリックして新しいボードを作成。
  2. 用途に合わせてテンプレート（例: Mind Map, Kanban）を選択。
  3. ツールバーから付箋やテキストを選んで配置開始。

## **5. 特徴・強み (Pros)**

* **圧倒的なテンプレート数**: コミュニティベースの「Miroverse」には5,000以上のテンプレートがあり、ベストプラクティスを即座に導入できる。
* **強力な外部連携**: JiraやAzure DevOpsとの双方向同期が可能で、付箋をタスクとして管理できるため、開発現場での親和性が高い。
* **オールインワン**: ブレインストーミング用ツール、作図ツール、タスク管理ツールをMiro一つに集約できる。
* **直感的な操作性**: 誰でも直感的に使えるUIでありながら、ショートカットキーなども充実しており熟練者にも使いやすい。

## **6. 弱み・注意点 (Cons)**

* **パフォーマンス**: オブジェクト数が数千を超えるような超巨大なボードでは、読み込みや描画が遅くなる場合がある。
* **学習コスト**: 機能が非常に多いため、初心者が全ての機能を把握するには時間がかかる。
* **オフライン利用**: 基本的に常時オンライン接続が必要で、完全なオフラインモードは提供されていない。
* **モバイル体験**: タブレットやスマホアプリも存在するが、複雑な編集作業はPC版に比べて行いにくい。

## **10. エコシステムと連携**

### **10.1 API・外部サービス連携**

* **API**: REST API, Web SDKが公開されており、ボード上のアイテム操作や外部システムとのデータ連携が可能。
* **外部サービス連携**:
  * **Jira / Azure DevOps**: 付箋とチケットの双方向同期。
  * **Slack / Microsoft Teams**: 通知の受け取りや、アプリ内でのボード展開。
  * **Google Workspace / Microsoft 365**: ドキュメントの埋め込みやSSO連携。
  * **Figma / Adobe XD**: デザインファイルの埋め込みや連携。

### **10.2 技術スタックとの相性**

| 技術スタック | 相性 | メリット・推奨理由 | 懸念点・注意点 |
|:---|:---:|:---|:---|
| **JavaScript / TypeScript** | ◎ | Web SDKはJS/TSベースであり、型定義も提供されているため開発体験が良い。 | 特になし |
| **React** | ◎ | Miro Web SDKはReactとの親和性が高く、カスタムアプリのUI構築が容易。 | 公式のUIコンポーネントライブラリ（Mirotone）の活用が推奨される。 |
| **Python** | ◯ | REST APIを利用して、データ分析やバッチ処理を行うのに適している。 | Web SDK（フロントエンド操作）は利用できない。 |
| **Node.js** | ◎ | REST APIのバックエンド処理やOAuth認証フローの実装に最適。 | 特になし |

## **11. セキュリティとコンプライアンス**

* **認証**: SSO (SAML 2.0), 2段階認証 (2FA)。Enterpriseではドメイン管理も可能。
* **データ管理**: 転送中および保存データの暗号化 (TLS 1.2+, AES-256)。Enterprise Guardによる高度なデータガバナンス。
* **準拠規格**: ISO 27001, SOC 2 Type II, SOC 3, GDPR, TISAX, NIST。

## **12. 操作性 (UI/UX) と学習コスト**

* **UI/UX**: 非常にスムーズなズーム・パン操作が可能で、付箋を貼る、線を繋ぐといった基本操作が心地よい。
* **学習コスト**: 基本操作は数分で覚えられるが、「スマートダイアグラム」や「開発連携」などの高度な機能を使いこなすには、公式のアカデミーやドキュメントを参照する必要がある。

## **13. ベストプラクティス**

* **効果的な活用法 (Modern Practices)**:
  * **テンプレート活用**: ゼロから作らず、Miroverseから目的に近いテンプレートを探してカスタマイズする。
  * **フレーム機能**: ボード内を「フレーム」で区切ることで、プレゼンテーションモードでのスライドショーやPDFエクスポートが容易になる。
  * **タグと検索**: 付箋にタグを付けたり、色分けのルールを決めることで、後からの検索や整理が効率化する。
* **陥りやすい罠 (Antipatterns)**:
  * **整理整頓不足**: 無限キャンバスだからといって無秩序に配置しすぎると、どこに何があるかわからなくなる（「フレーム」で整理すべき）。
  * **バックアップ忘れ**: 重要なボードは定期的にバックアップ（.rtbファイルとしてエクスポート）を取ることが推奨される（Enterprise以外では制限がある場合も）。

## **17. 総評**

* **総合的な評価**:
  * Miroは、単なるホワイトボードツールから「イノベーションワークスペース」へと進化しており、機能の豊富さ、外部連携の強さ、使いやすさのバランスが非常に高いレベルでまとまっている。特に開発チームやプロダクトチームにとっては、Jira連携や作図機能の充実度から、最も有力な選択肢である。
* **推奨されるチームやプロジェクト**:
  * アジャイル開発を行うソフトウェアチーム、プロダクトマネージャー、リモートワーク中心の組織、複雑な要件定義やシステム設計を行うプロジェクト。
* **選択時のポイント**:
  * 「何でもできる自由度」と「開発エコシステムとの統合」を重視するならMiro一択。逆に、Figmaを中心としたデザインフローがあるならFigJam、厳格な図解管理フローがあるならLucidsparkも検討に値する。
